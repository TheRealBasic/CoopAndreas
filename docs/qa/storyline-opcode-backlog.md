# Storyline opcode parity backlog

Generated by `python tools/opcode_audit.py` from storyline targets referenced in `docs/ROADMAP.md`.

## Audit process

1. Parse target storyline script file references from `docs/ROADMAP.md`.
2. Extract opcode usage from each target `scm/scripts/*.txt` file (numeric forms like `00BA:` + class/member forms like `Text.PrintNow(...)`).
3. Resolve class/member calls through `sdk/Sanny Builder 4/data/sa_sbl_coopandreas/sa_coop.json`.
4. Compare against `syncedOpcodes` in `client/src/COpCodeSync.cpp` (hex literals + `COMMAND_*` entries).
5. Triage missing opcodes by parity impact category.

## CI gate (opcode audit)

- Workflow job: `.github/workflows/ci.yml` → `opcode-audit`.
- Command:
  - `python3 tools/opcode_audit.py --output artifacts/storyline-opcode-backlog.md --max-missing 9`
- Expected output includes:
  - `Wrote /home/runner/work/.../artifacts/storyline-opcode-backlog.md`
  - `Missing vs syncedOpcodes: <count>`
- Failure criteria:
  - The job exits non-zero when `Missing vs syncedOpcodes` is greater than `9`.
  - On both pass and fail, the generated report is uploaded as the `opcode-audit-report` artifact for reviewer inspection.
